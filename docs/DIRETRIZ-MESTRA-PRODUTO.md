# DIRETRIZ MESTRA DE PRODUTO
Data: 2026-03-07
Versao: PR-DIRECTION-LOCK-01
Status: obrigatorio para qualquer PR futuro

## Regras inegociaveis
1. O produto final deve parecer jogo de verdade, nao app escolar gamificado.
2. O HTML inicial e anti-referencia estrutural. Nao orienta layout, UI, gameplay ou ritmo.
3. O projeto deve buscar:
- encantamento visual
- carisma
- fantasia luminosa
- direcao de arte cinematografica
- leitura inedita do universo SME
4. Qualquer elemento com cara de:
- wireframe escolar
- dashboard corporativo
- minigame generico
- prototipo didatico
deve ser refatorado ou removido.
5. O jogo deve parecer feito por um pequeno estudio excepcional, nao por experimento escolar.
6. O contexto SME e preservado como universo narrativo, nunca como formato de interacao.

## Antiobjetivo oficial
Falhas de produto:
- leitura longa com botao continuar como eixo
- quiz ou multipla escolha como nucleo
- tutorial textual ocupando o lugar da acao
- interfaces com cara de formulario

## Principios de experiencia
1. Mostrar > explicar.
2. Jogador age > jogador confirma.
3. Mundo reage > texto descreve.
4. Feedback audiovisual imediato em cada acao relevante.
5. Progressao dramatica curta e memoravel.

## Gate de aprovacao de PR (obrigatorio)
Todo PR deve declarar:
1. O que aumenta game feel.
2. O que remove passividade de leitura.
3. Como melhora carisma e impacto visual.
4. Se introduz risco de "cara escolar" e como mitiga.

Se nao responder aos 4 pontos, PR retorna para revisao.
